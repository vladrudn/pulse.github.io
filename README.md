# Pulse · опрацювання рапортів ГДО та МД

Демонстраційне робоче місце для швидкого опрацювання рапортів на:

- грошову допомогу для оздоровлення (ГДО);
- матеріальну допомогу для вирішення соціально-побутових питань (МД).

Демо: [https://vladrudn.github.io/pulse.github.io/](https://vladrudn.github.io/pulse.github.io/)

Проєкт демонструє сценарій, у якому діловод переходить від рапорту до готового
проєкту наказу з мінімальною кількістю кліків і без повторного введення однакових
даних.

## Основні можливості

- 52 повністю тестові особи без реальних персональних даних;
- пошук і фільтрація особового складу за підрозділом;
- створення запису безпосередньо з рядка військовослужбовця;
- ручне опрацювання рапорту з перевіркою номера та дат;
- запам'ятовування останньої дати рапорту;
- режим `+1` для автоматичного підставлення наступного номера рапорту;
- autocomplete раніше введених номерів і текстів підстав;
- до трьох довільних текстових підстав для МД;
- короткочасне підсвічування останньої опрацьованої особи у великому списку;
- імітація вхідних документів СЕДО з пакетним опрацюванням;
- тестові підстави СЕДО для матеріальної допомоги;
- відображення вже опрацьованих записів і захист від випадкового дублювання;
- підтвердження форми кнопкою або клавішею `Enter`;
- вибір проєкту наказу за датою;
- формування одного DOCX, який може одночасно містити пункти ГДО та МД;
- генерація наказу на основі `public/ШАБЛОН.docx` зі збереженням службового
  тексту, тризуба та налаштованих стилів документа.

## Сценарії роботи

### Ручне опрацювання

1. Відкрити розділ ГДО або МД.
2. Обрати особу в реєстрі.
3. Вказати номер і дату рапорту, а для МД за потреби додати підстави.
4. Натиснути `Опрацювати` або `Enter`.

Після збереження форма закривається, а рядок останньої особи тимчасово
підсвічується, щоб користувач не втрачав позицію у великому списку.

### Опрацювання із СЕДО

1. Відкрити `Вхідні СЕДО`.
2. Позначити один або декілька рапортів.
3. Вказати дату проєкту наказу.
4. Опрацювати вибраний пакет.

Номер, дата, особа та тестові підстави МД переносяться із запису СЕДО без
повторного ручного введення.

### Формування наказу

У розділі `Опрацьовано` потрібно вибрати наказ за датою та завантажити DOCX.
Якщо на цю дату опрацьовано ГДО і МД, обидва пункти формуються в одному
документі.

## Демонстраційні обмеження

- усі особи, рапорти та підстави є тестовими;
- імпорт реальних таблиць вимкнений;
- авторизація та серверна база даних відсутні;
- робочий стан зберігається лише в `localStorage` поточного браузера;
- різні браузери та пристрої мають незалежні дані;
- демоверсія не призначена для роботи з реальною службовою інформацією.

## Технології

- Next.js 16;
- React 19;

- статичний експорт Next.js;
- бібліотека `docx`;
- GitHub Pages і GitHub Actions.

## Локальний запуск

Встановлення залежностей виконується користувачем вручну:

```powershell
cd C:\MonoREPO\Pulse
npm install
npm run dev
```

Відкрити: [http://localhost:3000](http://localhost:3000)

Після першої інсталяції наступні запуски можна виконувати файлом `run_dev.bat`.
Лаунчер не встановлює залежності автоматично.

Перевірка перед публікацією:

```powershell
npm run lint
npm run build
```

## Публікація на GitHub Pages

Workflow `.github/workflows/deploy-pages.yml` автоматично:

1. запускається після push у гілку `main`;
2. встановлює залежності через `npm ci`;
3. створює статичну збірку командою `npm run build`;
4. публікує каталог `out` на GitHub Pages.

Для ручного оновлення:

```powershell
cd C:\MonoREPO\Pulse
git add .
git commit -m "Update Pulse demo"
git push origin main
```
