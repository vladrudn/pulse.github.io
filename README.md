# Імпульс · ГДО

Тестове робоче місце для опрацювання грошової допомоги для оздоровлення.
Інтерфейс використовує основні принципи ІКС «Імпульс», але адаптований під
швидке опрацювання рапортів ГДО.

## Можливості

- 50 повністю тестових військовослужбовців без можливості редагування;
- пошук і фільтрація за підрозділом;
- ручне створення запису з валідацією номера та дат рапорту;
- імітація реєстру вхідних документів СЕДО;
- пакетна обробка кількох окремих рапортів СЕДО;
- статуси `Чернетка` та `Опрацьовано`;
- формування і завантаження проєкту наказу у форматі DOCX;
- одна тестова роль діловода без авторизації;
- адаптивний інтерфейс для настільного комп'ютера і планшета.

Робочі записи зберігаються у `localStorage` браузера. Тестові особи й рапорти
СЕДО є статичними демонстраційними даними. Кожний браузер має власний незалежний
стан, тому ця версія не призначена для спільного одночасного ведення реєстру.

## Локальний запуск

Встановлення залежностей виконується користувачем вручну:

```powershell
cd C:\MonoREPO\Pulse
npm install
npm run dev
```

Відкрити: [http://localhost:3000](http://localhost:3000)

Після першої ручної інсталяції наступні запуски можна виконувати файлом
`run_dev.bat`. Лаунчер не встановлює залежності автоматично.

Перевірка перед публікацією:

```powershell
npm run lint
npm run build
```

## Розгортання на Vercel

1. Створити GitHub-репозиторій і додати в нього вміст папки `Pulse`.
2. Імпортувати репозиторій у Vercel.
3. Framework Preset має автоматично визначитися як `Next.js`.
4. Build Command: `npm run build`.
5. Додаткові змінні середовища для тестової версії не потрібні.

Під час наступного етапу `localStorage` можна замінити на PostgreSQL/Neon, не
змінюючи структуру тестових осіб, форми й пакетного сценарію.
